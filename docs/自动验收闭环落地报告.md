## B. 各脚本功能说明

### 1. `state-doctor.js` — 账号状态文件诊断

**路径**：`Dreamina/0.0.3/account-state/state-doctor.js`

**检查项**：

| 检查 | 级别 | 说明 |
|------|------|------|
| 四文件存在性 + JSON 格式 | ERROR | 任一文件不是数组则报错 |
| local ∩ registered 去重 | ERROR | 已注册账号还在待跑池 = 数据不一致 |
| local ∩ blacklisted 去重 | ERROR | 黑名单账号还在待跑池 = 会浪费资源重跑 |
| registered ∩ blacklisted 去重 | WARN | 语义矛盾，需人工确认 |
| local ∩ retry 去重 | WARN | retry 账号已回注但 retry 文件未清空 |
| accounts-done.txt 一致性 | WARN | done.txt 与 registered 计数差异过大 |
| retry / local 条目字段完整性 | ERROR/WARN | 缺少 password 则无法重跑 |

**退出码**：
- `0` = 无 ERROR（WARN 不阻断）
- `1` = 有 ERROR（数据不一致，需修复后再跑）

---

### 2. `result-doctor.js` — 批量结果文件诊断

**路径**：`Dreamina/0.0.3/account-state/result-doctor.js`

**检查项**：

| 检查 | 级别 | 说明 |
|------|------|------|
| batch-results/ 存在性 | WARN | 首次运行为正常 |
| accounts-done.txt 行格式 | WARN | 不含 @ 的行为格式异常 |
| 最近批次成功率 | WARN | 低于 80% 自动触发警告 |
| session-records/ 文件数 | WARN | 超过 500 建议归档 |
| local 账号池是否耗尽 | WARN | local=0 时提示补充 |

**附加参数**：
- `--last-batch`：打印最近 50 条失败原因分布

---

### 3. `smoke-register.js` — 单账号冒烟注册

**路径**：`Dreamina/0.0.3/smoke-register.js`

**Three-phase 结构**：

```
Phase 1: Preflight
  ├── state-doctor    (ERROR → 中止)
  ├── result-doctor   (WARN → 继续)
  └── config-doctor   (ERROR → 中止)

Phase 2: 注册执行
  └── 调用 Dreamina-register.js --headed --slow-mo 200

Phase 3: 结果断言
  └── 验证 results/latest/ 中最近结果 success=true + finalStage=account-delivery
```

**退出码**：
- `0` = SMOKE PASS
- `1` = 注册失败
- `2` = preflight 中止（ERROR 级别问题）

## C. 运行方式

### 日常维护（批次前推荐执行）

```bash
# 全量诊断（state + result doctor）
npm run doctor

# 仅账号状态检查
npm run state:check

# 仅结果文件检查（含最近批次失败原因）
npm run result:check -- --last-batch

# 软失败账号回注
npm run recycle
```

### 冒烟测试

```bash
# 标准冒烟（含 preflight，headless）
npm run smoke

# 可视模式冒烟（指定账号和代理）
node Dreamina/0.0.3/smoke-register.js --account-index 0 --proxy-index 5

# 跳过 preflight 直接跑注册
node Dreamina/0.0.3/smoke-register.js --skip-preflight --headless
```

### 推荐工作流

```
批次启动前：
  npm run state:check    → 确认账号状态无 ERROR
  npm run recycle        → 回注可用 retry 账号
  npm run batch          → 启动批量

批次结束后：
  npm run result:check -- --last-batch   → 查看失败原因分布
  npm run state:check                     → 验证状态文件一致性

框架修改后：
  npm run smoke                           → 冒烟验证主链路可用性
```

## E. 剩余风险

| 风险 | 处置 |
|------|------|
| `smoke-register.js` 依赖真实代理和邮箱服务，无法在离线环境运行 | 使用 `--skip-preflight` 可跳过 doctor，但注册本身仍需网络 |
| `result-doctor` 对 batch-summary-latest.json 路径有假设（可能随 batch 版本变化） | 路径不存在时降 WARN 处理，不阻断 |
| state-doctor 的 done.txt 一致性检查对老版本数据有误报 | 阈值设为 >10 条才触发，现有 923 条差异为历史遗留，已标注为 WARN |
| smoke-register 结果断言依赖 results/latest/index.json 格式，格式变化时断言会跳过 | 断言失败时降级为仅看 exit code |
