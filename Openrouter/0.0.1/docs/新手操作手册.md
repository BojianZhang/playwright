# OpenRouter 自动化 · 新手操作手册（小白版）

> 这份文档假设你**完全没接触过这个项目**，从零开始一步步带你跑起来。
> 只讲「网页控制台」这条最简单的路（在浏览器里点鼠标就能用），不需要你写代码。
> 命令行的高级玩法放在最后第 9 节，新手可以先不看。

---

## 1. 这个东西是干嘛的？

一句话：**它能自动帮你在 [openrouter.ai](https://openrouter.ai) 上批量注册账号、领取 API Key、绑定信用卡、充值**，整个过程不用你一个个手动点。

它会自动完成这一连串动作：

```
注册账号 → 收激活邮件点链接 → 登录 → 领取 API Key → 填账单地址 → 绑信用卡 →（可选）充值
```

你要做的只是：**准备好"原料"（邮箱、卡、代理 IP 等），把它们填进网页，然后点"开始"，剩下交给它。**

---

## 2. 开始之前，你要先准备好这 6 样"原料"

把它想象成做饭：下面这些就是食材，缺了某样就做不出某道菜。**前 3 样是注册必需的，后 3 样是"加卡/充值"才需要**，如果你只想注册拿 Key，后 3 样可以先不管。

| # | 原料 | 是什么 / 去哪弄 | 干嘛用的 | 必需？ |
|---|------|----------------|----------|--------|
| 1 | **打码服务 Key** | 去 [2captcha.com](https://2captcha.com) 或 [capsolver.com](https://capsolver.com) 注册并充点钱，拿一个 API Key | 注册时网站会弹"人机验证"（Cloudflare/hCaptcha），靠它自动破解 | ✅ 注册必需 |
| 2 | **邮箱服务 Key** | 去 [Firstmail](https://firstmail.ltd) 之类的邮箱服务买，拿一个 API Key | 注册要收激活邮件、点里面的链接，靠它自动读邮件 | ✅ 注册必需 |
| 3 | **AdsPower（指纹浏览器）** | 在你这台电脑上[下载安装 AdsPower 客户端](https://www.adspower.com)，打开它的"Local API" | 给每个账号一个独立"伪装身份"的浏览器，降低被网站风控的概率 | ✅ 强烈建议 |
| 4 | **代理 IP** | 找住宅代理服务商买（格式 `host:port:用户名:密码`） | 每个账号用不同的出口 IP，避免被网站发现"都是同一个人" | 加卡建议 |
| 5 | **信用卡** | 你要绑的支付卡（卡号、有效期、CVV） | 给账号绑卡 / 充值用 | 加卡时必需 |
| 6 | **账单地址** | 跟卡匹配的账单地址 | 绑卡时网站要填地址（影响过审） | 加卡时需要 |

> 💡 **新手建议**：第一次先只准备 1、2、3，目标设成"只注册拿 Key、不绑卡"，跑通了再加 4、5、6 去试绑卡。一口吃不成胖子。

## 3. 第一次启动（Windows 电脑）

### 3.1 装好基础环境（只需做一次）

1. **装 Node.js**：去 [nodejs.org](https://nodejs.org) 下载 **20 版本以上**，一路下一步装好。
   验证：打开 PowerShell，输入 `node -v`，能看到版本号（如 `v20.x`）就对了。

2. **装项目依赖**：在 PowerShell 里进入项目根目录，运行：
   ```powershell
   cd d:\project\playwright
   npm install
   npx playwright install chromium
   ```
   > 这一步会下载项目要用的"零件"和一个内置浏览器，第一次会慢一点，耐心等。

### 3.2 启动网页控制台

在 PowerShell 里运行（在项目根目录）：

```powershell
node Openrouter/0.0.1/web/server.js
```

看到类似 `listening on http://0.0.0.0:4317` 的提示，就说明启动成功了。

**这个窗口不要关**（关了控制台就停了）。然后打开浏览器，访问：

👉 **http://localhost:4317**

你就看到控制台界面了。左边一列是菜单，分成「运行 / 资源 / 系统」三组。

> 🛑 **想关掉它**：回到那个 PowerShell 窗口，按 `Ctrl + C`。

## 4. 跟着"部署引导"把原料填进去（核心步骤）

第一次打开网页，它通常会**自动跳到「部署引导」页**（如果没有，点左侧菜单 **系统 → 部署引导**）。

这个引导页把"必须配齐才能开跑"的东西做成了**一步一步的清单**，每一步都能当场测试。页面顶部有进度条，左边是步骤列表（打勾✓ = 已配好，叹号! = 还差）。

下面挨个说明每一步怎么填（顺序可能略有不同，按页面上的来）：

### ① 验证码（必需）
- **服务商**：选你买的那家（2Captcha 或 CapSolver）。
- **API Key**：粘贴第 2 节准备的打码 Key。
- 点 **添加 key**，再点 **查全部余额** 确认 Key 有效、有钱。

### ② 邮箱（必需）
- **服务商** 填 `firstmail`（或你用的服务），**API 地址** 一般保持默认。
- **API Key**：粘贴你的邮箱服务 Key。
- 点 **添加 key**。

### ③ AdsPower（强烈建议）
- 先**打开你电脑上的 AdsPower 客户端**，在它的「设置 → Local API」里确认 API 开着，地址一般是 `http://127.0.0.1:50325`。
- 在网页里 **Local API 地址** 填上面那个地址，**API 密钥** 一般**留空**（本机不用填）。
- 点 **保存并测试连接**，看到"连接成功"就对了。
  - ❗ 如果连不上：多半是本机走了系统代理。把地址里的 `local.adspower.net` 改成 `127.0.0.1`，或在系统里排除代理。

### ④ 访问令牌（可选，建议）
- 这是给控制台加个"密码"，防止别人连到你的端口偷走账号。
- **只在你自己电脑上本地跑，可以跳过**；如果这台机器开了公网访问，强烈建议设一个随机长字符串。
- 填好点 **保存令牌**（注意：保存后要**重启** `server.js` 才生效，之后访问都要带这个令牌）。

### ⑤ 代理 / IP（加卡建议）
- 把代理粘进去，**一行一个**，格式 `host:port` 或 `host:port:用户名:密码`。
- 点 **导入**。

### ⑥ 卡池（加卡必需）
- **每张卡最多用几次**：填个数字（比如 3）。
- 把卡粘进去，**一行一张**，格式按提示（一般 `卡号|月|年|CVV`，例 `4111111111111111|12|2028|123`）。
- 点 **导入卡**。

### ⑦ 账单地址（加卡需要）
- 把地址粘进去，一行一个，支持竖线/CSV/Tab 分隔。
- 点 **导入**。

### ⑧ AdsPower 环境（可选）
- **全新跑可以留空**——流水线会自动给每个账号建环境。
- 只有你想"复用已有的老环境续跑"时才填环境 ID。

全部配完后，看顶部进度条，显示 **"必需项已齐 · 已可开跑 🎉"** 就成功了。点右下角 **完成引导**，它会带你去「控制台」。

> 以后想改这些原料，不用再走引导，直接点左侧「资源」组里的对应页面（卡池 / 代理 / 账号 / 验证码 / 邮箱 …）就能管理。

## 5. 跑第一批账号（控制台页面）

进入左侧 **运行 → 控制台**。这是个分步表单，从上往下填：

1. **账号**：把要注册的账号粘进去，**一行一个**，格式通常是 `邮箱:密码`。
   （这些邮箱要是你邮箱服务里真实能收信的邮箱。）

2. **选引擎 + 引用配置**：选用哪个引擎跑、引用哪套"引擎配置"。新手用默认的即可。
   - 引擎专属的参数（要不要绑卡、最多换几次环境等）在左侧 **运行 → 引擎配置** 里调；
   - 每个环节的业务参数（Key 名字、卡用几次、充多少钱）在 **运行 → 环节策略** 里调。
   - **新手不用动这两个，先用默认。**

3. **流程开关**：勾选这次要做哪些环节。
   - 只想**注册拿 Key**：把"加卡 / 充值"的勾**去掉**。
   - 想**绑卡**：勾上加卡。想**充值**：勾上充值（⚠️ 充值是**真扣钱**的，确认无误再勾）。

4. **并发数**：同时跑几个账号。**新手填 1**（一个一个跑，稳）。机器好、代理足了再往上加。

5. 确认无误，点 **开始**。

页面会**实时显示进度**：每个账号跑到哪一步、成功还是失败、卡池用了几张、烧了几个环境，都能看到。

> ⏱️ 第一次跑，盯着看完整一个账号的全过程，确认每一步都正常，再放心批量跑。

## 6. 跑完了，去哪看结果？

| 我想看… | 去这个页面 |
|---------|-----------|
| 这次跑的实时进度 | **运行 → 控制台**（跑的时候就在这看） |
| 历史上每一批跑了什么 | **运行 → 运行历史**（点进去看每批详情） |
| 所有成功的账号 + Key 汇总、导出 | **运行 → 结果聚合**（可勾选导出 / 删除） |
| 整体健康度、今天跑了多少 | **运行 → 总览** |
| 为什么老失败、卡在哪个环节 | **系统 → 失败分析**（有漏斗图和原因排名 + 建议） |
| 某次具体哪里报错了 | **系统 → 诊断 / 排查** |

成功的账号和 Key 会自动存档，可以在「结果聚合」页勾选后**导出**成文本。

## 7. 常见问题速查

| 现象 | 多半是什么原因 | 怎么办 |
|------|--------------|--------|
| 网页打不开 / 转圈 | `server.js` 没在跑，或端口不对 | 回 PowerShell 看窗口还在不在；确认地址是 `http://localhost:4317` |
| AdsPower"连接失败" | 客户端没开 Local API，或走了系统代理 | 打开 AdsPower 客户端、开 Local API；地址用 `127.0.0.1` 不要用 `local.adspower.net` |
| 注册老卡在验证码 | 打码 Key 没钱 / 失效 | 去「资源 → 验证码」点查余额，没钱就充值或换 Key |
| 收不到激活邮件 | 邮箱 Key 失效，或邮箱本身收不到信 | 检查邮箱服务 Key；换能正常收信的邮箱 |
| 好卡也一直绑不上 / 报 502 | 这个**环境被网站风控了**（不是卡坏） | 多准备**≥2 个干净环境**，系统会自动换；单环境只能你手动换 IP/环境重试 |
| 卡填了一半 / 只填上前几位 | 没用真实指纹浏览器接管 | 确认选了 AdsPower 并配好了环境 |
| "环境 烧/总"数字一直涨变红 | 这批代理/环境出口被风控了 | 停下来，换一批干净的代理 + 环境再跑 |

> 更详细的"加卡/换环境/风控"排障，看同目录的 [加卡与环境-操作指南.md](加卡与环境-操作指南.md)。

## 8. 安全须知（重要，务必看）

- **卡号、各种 API Key、密码都是敏感信息**。项目已经设置好不把它们提交到 Git，但你自己也别把这些文件发给别人。
- 如果这台机器**能被公网访问**，一定要在引导第④步**设访问令牌**，否则任何人连上你的端口就能拿走你的账号和 Key。
- **充值是真花钱的**，跑批前务必确认"流程开关"里充值的勾是不是你想要的。

## 9.（进阶）命令行批量模式 —— 新手可跳过

网页控制台用的是「纯 Playwright」引擎。如果你要**大批量 + 更强的过风控加卡能力**，项目还有两种命令行模式（需要装 Python）：

| 模式 | 启动命令（在 `Openrouter/0.0.1/selenium-e2e/` 目录下）| 特点 |
|------|------|------|
| **混合接力**（推荐批量） | `python hybrid_run.py` | 先用 Playwright 注册/取 Key/绑地址，再用 Selenium 的 "Fix C" 加卡（更能过 Stripe 风控、可并发） |
| **纯 Selenium** | `python run.py` | 全流程都用 Selenium 跑，加卡也用 Fix C |

这两种模式有很多命令行参数（并发数、是否充值等），详见：
- [../ARCHITECTURE.md](../ARCHITECTURE.md) —— 三种运行模式总览
- [../selenium-e2e/ENTRYPOINTS.md](../selenium-e2e/ENTRYPOINTS.md) —— 命令行入口地图
- [DEPLOY.md](DEPLOY.md) —— 部署到 Linux 服务器
- [集群部署手册.md](集群部署手册.md) —— 多台机器一起跑、中心机汇总

## 附：最快上手路线（TL;DR）

1. 装 Node 20+ → `cd d:\project\playwright` → `npm install` → `npx playwright install chromium`
2. `node Openrouter/0.0.1/web/server.js`，浏览器开 **http://localhost:4317**
3. 跟着「部署引导」填：验证码 Key、邮箱 Key、AdsPower（前 3 个就够注册）
4. 进「控制台」：粘账号 → **去掉加卡的勾** → 并发填 1 → 点开始
5. 跑通一个账号后，再加代理/卡/地址去试绑卡，并发慢慢调大
6. 结果去「结果聚合」看和导出
