# OpenRouter 集群部署操作手册

主机(中心机,看汇总数据) + 多台子机(跑任务,自动上报)的配置步骤。

---

## 总览(顺序很重要)

1. **先定一个 token**(全集群共用一个,随机串)。
2. **配置并启动主机**(中心机)。
3. **每台子机配置 centralUrl + 同一 token,启动** → 自动注册并推送数据给主机。
4. 在主机打开 results 页看全集群实时汇总。

> 说明:子机找不到主机会每 30s 自动重试,所以"主机先起"只是推荐、不是强制;但**建议先起主机**。

---

## 第 0 步:每台机器的前置(主机/子机都要做一次)

```bash
# 1) Node 18+(建议 20+);进入仓库根目录
cd /path/to/playwright
npm install
npx playwright install chromium
npx playwright install-deps chromium      # Linux 需要;Windows 跳过
# 2) (推荐)装真实 Chrome,过 Turnstile 更稳;没有也会自动回退内置 Chromium
```

每台机器都要有密钥文件 `Openrouter/0.0.1/config/config.local.json`(**不进 Git**):
```json
{
  "captcha": { "provider": "2captcha", "apiKey": "你的2captcha_key" },
  "mailbox":  { "apiKey": "你的firstmail_key" }
}
```

---

## 第 1 步:生成 token(只做一次,全集群共用)

随便用下面任一种生成一个长随机串:
```bash
node -e "console.log(require('crypto').randomBytes(24).toString('hex'))"
```
得到类似 `5ee3b8d2....`,**记下来**,主机和所有子机都用这一个。

---

## 第 2 步:配置并启动【主机/中心机】

编辑主机的 `Openrouter/0.0.1/config/config.local.json`,加上 `security`:
```json
{
  "captcha": { "provider": "2captcha", "apiKey": "..." },
  "mailbox":  { "apiKey": "..." },
  "security": {
    "token": "第1步生成的token"
  }
}
```

启动主机:
```bash
node Openrouter/0.0.1/web/server.js
```
看到日志 `🔒 访问令牌已启用` 即正常。访问:
- 本机:`http://localhost:4317`
- 局域网:`http://主机内网IP:4317`
- 公网:`http://主机公网IP或域名:4317`(需公网可达,见下方"主机对外可达")

> 主机自己不跑任务也行,只当"看板 + 数据汇总仓库"。

---

## 第 3 步:配置并启动【每台子机】

编辑子机的 `Openrouter/0.0.1/config/config.local.json`,加 `security.token`(同一个)+ `cluster.centralUrl`(指向主机):
```json
{
  "captcha": { "provider": "2captcha", "apiKey": "..." },
  "mailbox":  { "apiKey": "..." },
  "security": { "token": "和主机同一个token" },
  "cluster":  { "centralUrl": "http://主机地址:4317" }
}
```
`centralUrl` 填**子机能访问到的主机地址**:
- 同一局域网 → `http://主机内网IP:4317`
- 异地/公网 → `http://主机公网域名或IP:4317`(或 `https://你的域名`)

启动子机:
```bash
node Openrouter/0.0.1/web/server.js
```
看到日志 `📡 自动注册并推送结果到中心机: ...` 即表示已在向主机上报。子机在自己本机也能看自己的数据:`http://localhost:4317/results.html`。

> 子机端口可不同:`OPENROUTER_WEB_PORT=4318 node ...`。

---

## 第 4 步:在主机看全集群汇总

主机浏览器打开(首次带 token,自动记住):
```
http://主机地址:4317/results.html?token=你的token
```
- 自动每 5s 刷新,所有子机的成功账号实时汇总、去重展示;
- 下方"数据源"能看到每个来源(本机 / pull / push(节点名))各多少条;
- 可一键 复制 / 下载 .txt / .json。

---

## 主机对外可达(异地子机要用)

子机要能连到主机。三选一:
1. **同局域网**:直接用主机内网 IP,无需额外设置(注意主机防火墙放行 4317)。
2. **VPN(Tailscale,推荐异地)**:主机+子机都装 Tailscale,centralUrl 用主机的 Tailscale IP。不暴露公网、最安全。
3. **公网**:主机有公网 IP/域名(云服务器,或路由器端口转发),**建议套 HTTPS**(Nginx/Caddy 反代)。子机 centralUrl 用该公网域名。

Windows 防火墙放行(管理员 PowerShell,一次):
```powershell
New-NetFirewallRule -DisplayName "OpenRouter 4317" -Direction Inbound -Protocol TCP -LocalPort 4317 -Action Allow
```

---

## 安全选项(可选,按需加在主机 security 里)

```json
"security": {
  "token": "...",
  "allowHosts": ["你的域名"],            // 只接受用此域名访问,挡裸IP扫描
  "allowIps":   ["公司A出口IP", "公司B出口IP"], // 只让这些网络"看"results(公网出口IP,非内网192.168.x)
  "trustForwardedFor": true,             // 套了 Nginx 反代才设 true
  "gateStatic": false                    // true=没token连页面都打不开
}
```
**重要**:`allowIps` 只限制"查看/聚合"路由;**子机的注册/推送不受 allowIps 限制**(任意 IP 带 token 即可推),所以"公网主机 + 公司IP才能看 + 异地子机照常推"可同时成立。
> 公网部署务必加 HTTPS(明文 token 会被嗅探)。

---

## 常见问题

- **子机连不上主机**:检查 centralUrl 是否子机可达、主机防火墙是否放行、token 是否一致。
- **results 页弹"需要 token"**:用 `?token=` 网址访问一次,或弹窗里粘 token。
- **公司内网看不了(公网主机+开了allowIps)**:allowIps 要填**公司公网出口 IP**(不是 192.168.x);套反代要开 `trustForwardedFor`。
- **子机掉线**:主机 90s 后自动标记离线;**已推送的数据保留不丢**。
- **换 token**:改所有机器的 `config/config.local.json` 重启,浏览器重新输一次。

## 进程常驻(可选,pm2)
```bash
npm i -g pm2
pm2 start "node Openrouter/0.0.1/web/server.js" --name openrouter
pm2 save && pm2 startup
```
