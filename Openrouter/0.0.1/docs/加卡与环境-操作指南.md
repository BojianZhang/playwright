# OpenRouter 加卡 / 充值 · 操作与排障指南

> 本文把「加卡」这条链路串起来：**指纹浏览器 → CDP 接管 → 填卡引擎 → 账单判定 → 撞风控自动换环境**。
> 子系统各自的细节见：[browser-provider/README](browser-provider/README.md)（指纹浏览器）、[automation-driver/README](automation-driver/README.md)（自动化驱动）、[card-fill/extension/README](billing/card-fill/extension/README.md)（扩展填卡引擎）。

---

## 1. 一条链路（数据怎么流）

```
账号 → [指纹浏览器 provider].start(envId) ──CDP ws/debugPort──┐
                                                              ▼
               browser-provider 用 connectOverCDP 接管同一浏览器（穿 Stripe 跨域 iframe 的前提）
                                                              │
                  stages.js 跑 注册→登录→取Key→账单                │
                                          │                     │
                  账单阶段：[填卡引擎] 填 Stripe 卡表单 ───────────┘
                                          │
              点 Save → 必要时 2captcha 解 hCaptcha → 判定结果
                                          │
        ┌──────────────┬──────────────────┼─────────────────┬─────────────────┐
     success        card-bound         declined         server-error        (无弹窗等)
   充值成功,计用量   仅加卡不扣费,不计用量  真·卡被拒→踢卡   环境疑似被风控→换干净环境  其它瞬时错→重试
```

**为什么必须 CDP 接管而不是裸 `playwright test`**：Stripe 卡字段在 `js.stripe.com` **跨域 iframe** 里，只有接管真实指纹浏览器（AdsPower 等）才穿得过去；裸浏览器会被判 bot（弹九宫格 hCaptcha）。实测接管能 16/16 位填满。

## 2. 三个可切换维度（都在网页表单选）

| 维度 | 选项 | 默认 | 说明 |
|---|---|---|---|
| **指纹浏览器** `browserProvider` | none / adspower / bitbrowser / dolphin / gologin / hubstudio / morelogin / multilogin / vmlogin | none（原生 Playwright） | 选 `adspower` = 原 AdsPower 接管行为。除 adspower 外本机未实测，需你侧验证本地 API |
| **填卡引擎** `cardFillEngine` | playwright / osinput / extension / selenium / api（可逗号链做兜底） | playwright | playwright 走 `page.frames()` 直填 iframe，最稳。`osinput` 走系统按键（**需 `OPENROUTER_OSINPUT_OK=1` 显式开**，否则会把卡号发到前台窗口） |
| **环境ID池** `browserEnvIds` | 一行/逗号一个环境 id | — | 每账号分一个（自带代理+指纹）。撞风控时自动在池内换干净的，见 §4 |

> 密钥（2captcha / firstmail / 安全 token）只走 **环境变量** 或 **`config/config.local.json`**（gitignore），**绝不进 `config.json`**。

## 3. 账单判定：每种结果意味什么

判定逻辑在 [stages.js](stages.js) `runBillingFlow` / `detectOutcome` / Save 重试循环：

- **success**：充值真扣费成功 → 卡计一次用量。
- **card-bound**：弹窗推进到购买页 = 卡已存上（未扣费）→ 卡**不**计用量、保持可用。
- **declined**：**真·卡被拒**（余额不足 / 卡号错 / CVC 错 / 过期 / 明确 declined）→ 踢卡（status=declined）。
- **server-error**（`BILLING_SERVER_ERROR`）：付款反复 5xx / 网关错 = **环境疑似被 Stripe Radar 风控**，不是卡的问题 → 卡 report 为 `error`（**不踢卡、保持 active**），并触发换环境（§4）。
- 卡在验证码/无报错文案：**不再**当成功（旧逻辑会误判 card-bound），按未完成重试，仍不行收口为 server-error。

**关键认知（改重试逻辑必看）**：
- `502 / "try again"` 是**瞬时错**，不等于卡被拒 —— Save 循环最多重试 4 次；循环顶 `detectOutcome` 对瞬时错**绝不**判 declined（否则会冤枉好卡）。
- 撞风控的细码在 `last.reason === 'BILLING_SERVER_ERROR'`，**不是** `last.code`（那是笼统的 `BILLING_FAILED`→abort）。

## 4. 撞风控自动换环境

某环境被风控后，反复在它上面试卡只会白烧卡。系统会：**标记该环境 burned（本次 run 全局规避）→ 换一个干净环境重试该账号**。

实现：[browser-provider/env-pool.js](browser-provider/env-pool.js)（基于同步原子锁）+ [Openrouter-job-runner.js](Openrouter-job-runner.js) 每账号重试循环。三条并发不变量（经对抗式复审踩出）：

1. **绝不裸开**：环境池满（其它 worker 占着）时不拿当前环境硬开 —— 那会和持锁 worker 双开同一指纹环境、谁先结束就关掉对方。改为退避重租，超 `maxEnvPoolWaits` 收口为 `ENV_POOL_BUSY`。
2. **保留亲和**：普通重试（换代理/重新登录）回**同一**热身好的环境，不被挤走。
3. **底线**：`markBurned` **绝不烧最后一个**干净环境，防一次（可能误判的）风控把整池烧光、后续账号全军覆没。

**调参**（环境变量优先，也可加到 `config.json` 的 `batch.*`）：

| 旋钮 | 默认 | 含义 |
|---|---|---|
| `OPENROUTER_MAX_ENV_ROTATIONS` / `batch.maxEnvRotations` | size-1 | 每账号最多换几次环境（仍受 `batch.maxAttempts` 硬顶） |
| `OPENROUTER_MAX_ENV_POOL_WAITS` | 6 | 环境池满时连续退避几次后放弃（按 `ENV_POOL_BUSY` 收口） |

**网页统计格「环境 烧/总」**：实时显示本轮烧了几个环境，烧>0 标红 —— 看到就该补干净环境了。

## 5. 排障

| 现象 | 多半原因 | 怎么办 |
|---|---|---|
| 好卡也 502 / "unable to authenticate" | **环境被烧**（Stripe Radar velocity ban，常因同环境反复失败累积） | 换**干净**环境；多环境时系统会自动换，单环境只能你手动换 |
| 「环境 烧/总」一直涨、最后 `ENV_POOL_BUSY` 或全账号失败 | 整批环境/代理出口被风控 | 停批，换一批干净环境+代理再跑 |
| `REGISTER_THREW`（邮箱字段超时） | 环境残留了别的账号登录态，`/sign-up` 被重定向 | 已自动 `detectSession`+`clerkSignOut` 处理；仍异常则清环境登录态 |
| 填卡只填上前几位 / `PLAYWRIGHT_FILL_INCOMPLETE` | 没接管真实指纹浏览器（裸浏览器穿不过 Stripe iframe） | `browserProvider` 选 adspower 等并配好 envId |
| 卡号被发到终端/前台而非表单 | 用了 `osinput` 引擎但没显式开 | 设 `OPENROUTER_OSINPUT_OK=1` 且确保浏览器在前台；或就用默认 `playwright` |

**单环境的局限**：只有 1 个环境时无法轮换（撞风控即 abort，卡不踢、保持 active）。要发挥自动换环境，准备 **≥2 个干净环境**。

## 6. 安全约束（务必遵守）

- `config/config.local.json`（含 2captcha/firstmail key、安全 token）、`data/card-pool.json`（含完整卡号/CVC）、`accounts.txt`、`order_*.txt`、`tests/test-*.spec.ts`、`_*.js`（诊断脚本，含真实卡号/账号）—— **全部已 gitignore，绝不提交**。
- 对外快照（卡池 UI/SSE）一律脱敏（卡号→`••last4`，不含 CVC）。
- 提交只提交**代码**，且仅在明确要求时。

---

## 7. 仍需真机验证的（如实）

- **多环境真实轮换**（e0 烧→换 e1 成功）需 **≥2 个干净环境**才能现场跑通；单环境路径已离线验证与今天逐位一致。
- **bitbrowser / dolphin / gologin** 按官方本地 API 实现、本机未实测；**hubstudio / morelogin / multilogin / vmlogin** 为脚手架，均需你侧用真实本地 API 验证（字段名对照各家文档）。
- 填卡引擎 `selenium`（经 chromedriver 接管 debugPort）需本机装 selenium + 对应 chromedriver。
