# OpenRouter 账号自动化系统 — 说明文档 · 项目架构 · 功能总览

> 本文是 `Openrouter/0.0.1/` 的**总入口文档**(给接手者/运营者读的一份)。
> 偏架构与边界看 [ARCHITECTURE.md](ARCHITECTURE.md);偏新手上手看 [docs/新手操作手册.md](docs/新手操作手册.md)。
> 各子目录的 `README.md` 是该模块的**边界说明**,改代码前先读对应那份。

## 1. 这是什么

一套**批量做 OpenRouter 账号**的自动化系统:接管 [AdsPower](https://www.adspower.net) 指纹浏览器,逐账号完成

> **注册 → 登录 → 取 API Key → 绑账单地址 → 绑卡(Stripe) → 充值**

并提供一个零依赖的 **Node Web 控制台(端口 `:4317`)** 做配置、批量下发、实时进度、结果聚合、资源管理与故障分析。支持 **Windows / macOS / Linux**,以及**多机集群**(中心机聚合 + 子机自动注册推送)。

**设计铁律(贯穿全系统):**
- **每账号一个干净 AdsPower 环境,跑完即删** —— 账号状态互不串味。
- **关键节点必须真成功才算成功**:`auth==ok && key 取到 &&(要加卡则 card-bound)&&(要充值则 purchase==success)`,任一不达标判失败,不做糊涂账。
- **碰钱的地方多层防重**:取 Key / 绑卡 / 充值都有去重信号,续跑**绝不重建 Key、不重绑卡、不重复扣款**。
- **反检测边界**:绑卡用 **Fix C 原生 CDP**(脱 chromedriver),这是项目的核心壁垒,非必要不动。

## 2. 功能总览(一眼看全)

| 类别 | 能力 |
|---|---|
| **账号全流程** | 注册(过 Turnstile)、魔法链接邮箱验证、登录(过 factor-two 邮箱 OTP)、关 onboarding 浮层、取 API Key、绑账单地址、Fix C 绑卡、充值 |
| **四种运行模式** | 纯 Selenium / 混合(推荐)/ Web-Playwright / Split 分流跨引擎衔接 |
| **加卡抗风控** | 同会话切代理 IP 重试、连续 server-error 早停、hCaptcha 换卡、卡表单超时刷新、按 BIN 当日限额摊匀、浏览器崩溃就地重启 |
| **断点续跑** | 逐阶段 checkpoint,被杀/重启后只补没做完的步,复用已登录环境/已建 Key/已绑卡 |
| **钱与卡池** | 卡池单一来源(一卡可绑多号)、真实充值开关(默认 dry-run 零成本)、按卡容量原子预留闸、充值台账、拒付原因分类 |
| **失败处置** | 失败归因(到哪一步因何失败)、坏邮箱登记、被封号永久跳过、按失败类型配恢复方案、自动重试失败号 |
| **Web 控制台** | 24 个页面:批量下发向导 + 实时监控 + 运行历史/详情 + 结果聚合(含改密/取新Key)+ 全部资源池管理 + 失败分析 + 系统健康 + 部署引导 |
| **集群与运维** | 多机派发与结果聚合、20+ 命令行运营工具、跨平台一键安装/启动 |

## 3. 快速开始(说明文档)

### 3.1 先备齐 6 样"原料"
1. **验证码服务**:2captcha 或 capsolver 的 API Key(过 Turnstile / hCaptcha)。
2. **邮箱服务**:Firstmail API Key(读注册魔法链接 + 登录验证码)。账号文件第二列是**邮箱密码**。
3. **AdsPower**:本机装好并开启 **Local API**(默认 `http://127.0.0.1:50325`),客户端保持运行。
4. **代理池**:每个新环境必须配代理;**优先 socks5**(实测 http 那批易被 AdsPower 报 Check Proxy Fail)。
5. **卡池**:Stripe 可用卡(`卡号|MM/YY|CVC`)。一卡可绑多号(实测约 10),建议**多 BIN/多发卡行**以绕开 Radar velocity 墙。
6. **账单地址**:加卡用(可由系统按州随机生成,或导入)。

### 3.2 安装与启动
| 平台 | 安装 | 启动 |
|---|---|---|
| **Windows** | 双击 [install.bat](install.bat) | 双击 [start.bat](start.bat) |
| **macOS/Linux** | `bash install.sh` | `bash start.sh` |

安装脚本自动装 Python/Node + 依赖 + 复制配置模板;启动脚本拉起 `web/server.js` 并打开浏览器到 `http://localhost:4317`。

> ⚠️ Windows 中文 `.bat` 有多字节解析坑 → `install.bat`/`start.bat` 是**纯 ASCII 瘦启动器**,只负责调用 UTF8-BOM 的 `.ps1`,别改回中文 `.bat`。

### 3.3 首次配置(两选一)
- **推荐**:打开控制台 → 首次自动进 **部署引导 `/setup`**,分步把 验证码 / 邮箱 / AdsPower / 访问令牌 / 代理 / 卡池 / 地址 配齐,每步可当场测试。
- **手动**:编辑 [config/config.local.json](config/config.json)(密钥,gitignore)后重启服务。

### 3.4 跑一批
- **网页**:`/console` 6 步向导(贴账号 → 设并发 → 选方案 → 选引擎 → 确认 → 看实时进度)。
- **命令行(混合,最稳)**:
  ```bash
  cd selenium-e2e
  python hybrid_run.py --accounts accounts.hybrid.txt --proxies proxies.local.txt --op-pw '统一密码' --concurrency 3 --do-purchase
  ```
- **命令行(纯 Selenium)**:
  ```bash
  python run.py --accounts accounts.local.txt --proxies proxies.local.txt --do-key --do-card --unified-pw '统一密码'
  ```

> **首次真扣款前**:先用默认 **dry-run**(不开 `--real-charge`)把全流程走通(到充值步不真点),确认无误再开真实充值,并先 `--charge-count 1` 真扣一笔验证。

## 4. 项目架构

### 4.1 三引擎 + 共享库 + 数据层
```
0.0.1/
├── playwright/        引擎① Node/Playwright 编排(注册/登录/取Key/绑地址 + 反检测/打码)
├── selenium-e2e/      引擎② Python/Selenium(Fix C 加卡 + 混合编排 + 纯Selenium全流程 + 充值)
├── web/               引擎③ Node Web 控制台(:4317;React SPA + 零依赖 server.js)
├── billing/           共享库:卡池 / 多引擎填卡 / 充值台账 / 地址生成
├── browser-provider/  共享库:指纹浏览器环境 / 代理池抽象(adspower/…)
├── automation-driver/ 共享库:自动化驱动抽象(playwright/puppeteer/selenium)
├── config/            非密配置 config.json + 密钥 config.local.json(gitignore)
├── data/              数据层:卡池/账号/台账/策略 JSON(★敏感,gitignore)
└── docs/ + *.md       文档(本文 + 架构 + 操作/部署手册)
```

| 引擎 | 语言 | ✅ 负责 | ❌ 不负责 |
|---|---|---|---|
| **① Playwright 编排** `playwright/` | Node | 注册/魔法链接登录/取Key/绑地址;Turnstile/hCaptcha 打码;反检测 stealth | 加卡躲检测(交②Fix C);环境/代理(交 browser-provider) |
| **② Selenium 加卡** `selenium-e2e/` | Python | **Fix C 原生CDP绑卡**、混合编排、纯Selenium全流程、充值 | — |
| **③ Web 控制台** `web/` | Node | HTTP/SSE/表单 → 驱动引擎批量跑;集群聚合 | 自身不做页面自动化 |

共享库(`billing`/`browser-provider`/`automation-driver`)是**叶子库**,不反向依赖引擎;`data/` 是纯数据 + 读写模块。详见 [ARCHITECTURE.md](ARCHITECTURE.md)。

### 4.2 四种运行模式
| 模式 | 入口 | 链路 | 绑卡方式 | 取Key 强度 | 何时用 |
|---|---|---|---|---|---|
| **纯 Selenium** | `python run.py` 或 web `/api/run` | 全程 Selenium | Fix C(`fill_mode=cdp`) | 较弱(新版向导是硬伤) | 单引擎简单批 |
| **混合**(推荐) | `python hybrid_run.py` | ① 注册/取Key/绑地址 → 断开不关浏览器 → ② Fix C 加卡 | **Fix C** | 强(PW 强取 Key) | 批量首选,最稳 |
| **Web-Playwright** | web `POST /jobs` | ③ → ① 全程 Playwright | Playwright 填卡(串行) | 强 | 网页直接跑、不要 Fix C 时 |
| **Split 分流** | web `/api/run`(engine=split) | 两引擎并跑 + **第二轮交叉衔接** | 视落点 | 容错最强 | 想自动兜底跨引擎 |

> **Split 跨引擎衔接**:纯 Sel 取 Key 进不去 → 转混合;混合加卡卡 hCaptcha → 转纯 Sel。每号两引擎各最多一次。

### 4.3 依赖方向(单向)
```
web/ ─┐
       ├─→ playwright/ ─→ billing/ ─┐
selenium-e2e/(子进程拉 playwright/hybrid-pw-stage.js)   ├─→ data/
                       └─→ browser-provider/ ───────────┘
                       └─→ automation-driver/
```

## 5. 端到端自动化流程与关键技术

逐账号流水线(`selenium-e2e/pipeline.py` / `hybrid_run.py`,Web 经 `engine-runner.js` spawn):

```
建环境(配代理+随机指纹)
  → 会话检测(已登录?跳过注册)
  → 注册:填表 + 过 Turnstile + Firstmail 魔法链接验证
  → 登录:#password-field + 过 factor-two 邮箱 OTP
  → 关 onboarding 浮层(绝不关账单弹窗)
  → 取 API Key(新号向导内抓 / 老号工作台 New Key)
  → 绑账单地址(跨域 iframe,填完失焦)
  → 绑卡:Fix C 原生 CDP 可信填卡 + Save(撞 Radar 则切IP/换卡/刷新重试)
  → 充值 Purchase(可 dry-run;真扣前 checkpoint 去重)
  → 删环境(仅真正 card-bound 才删,给不上则保留待续跑)
```

| 关键技术 | 说明 | 位置 |
|---|---|---|
| **Fix C 原生 CDP 绑卡** | 坐标定位 → kill chromedriver 脱离 → CDP `Input.dispatch` 注入**可信输入**(isTrusted=true)→ 绕过 Stripe JS 注入检测。坐标用 CSS 像素无需 DPR 换算。 | `cardbind/fixc_core.py` |
| **纯 Selenium 过 Turnstile** | CDP Fetch 拦 `api.js` 把 wrapper 拼进正文(等价 Playwright route),必须在**导航到注册页前**启动。 | `services/cdp_fetch.py::TurnstileApiPatcher` |
| **注册=魔法链接,登录=验证码** | 注册邮箱验证是 Clerk `verify` 魔法链接;登录的二次校验(factor-two)才是 6 位 OTP。 | `services/firstmail.py` |
| **factor-two OTP 需真实邮箱密码** | 读 OTP 走 Firstmail `messages/latest`,body 要 `email+password`,密码错回 401 → 登录/取Key 工具必须拿到该号**当前邮箱密码**。 | `steps/steps_auth.py::_enter_otp` |
| **hCaptcha** | 2captcha 走**账号同一代理 IP**解(token IP=会话IP 防 502);解不动则**换卡**(同会话换不同 BIN),不来回切 IP。 | `steps/steps_billing.py` |
| **绑卡核验权威信号** | 刷新 Credits 页看掩码卡尾号 / Auto Top-Up;onboarding 浮层在页时不靠弱信号误判已绑。 | `steps/steps_billing.py::_card_attached` |
| **Key 抢救** | PW 抓到 key 当场打 `APIKEY_CREATED <sk-or-...>`,Node 即便超时被杀,Python 也能从 stderr 捞回,绝不留孤儿 Key。 | `hybrid-pw-stage.js` / `hybrid_run.py` |

## 6. Web 控制台功能(`:4317`,24 个页面)

> React SPA(`web/src/`)产物构建进 `web/public/`,由零依赖 `web/server.js` 托管(含 SPA fallback)。
> 左侧导航分 **运行 / 资源 / 系统** 三组。鉴权:设了访问令牌后所有页面/接口需带 token。

### 6.1 运行类
| 路由 | 页面 | 功能 |
|---|---|---|
| `/` | 总览仪表盘 | 7 天趋势 + KPI(成功率/卡池/错误)+ 卡池分布 + 近期运行 |
| `/console` | **控制台** | 6 步下发向导(账号/并发/方案/引擎/确认/SSE 实时流)+ 常驻监控面板(卡池/台账/账号状态/错误) |
| `/flow` | 执行流程 | 可视化每引擎的步骤时间线 + 元素维护 + 就地切走法预设(只读不动执行逻辑) |
| `/runs` · `/runs/:jobId` | 运行历史 · 详情 | 过往任务列表(排序/筛选/导出);单次下钻:汇总 + 成功表 + 失败原因表 + **按失败主因批量恢复/续跑** |
| `/results` | **结果聚合** | 多节点聚合去重 + 搜索 + 导出;**改密**(四列:邮箱原/现、OR 原/现)、**获取新Key**(已绑卡但缺 Key 的号登录后建 Key)、卡池只读 |
| `/analysis` | 失败分析 | 漏斗 + 环节失败排名 + 智能分类(外部 vs 可优化)+ IP/卡战绩 + 趋势(按引擎/天数筛) |

### 6.5 后端 API(`web/server.js`,要点)
- **下发/执行**:`POST /jobs`(Playwright)、`POST /api/run`(selenium/hybrid/split)、`POST /api/run/resume`(续跑)、`POST /api/jobs/stop`、`POST /api/dispatch`(集群派发)。
- **实时进度**:`GET /events`(SSE,带 `Last-Event-ID` 断线重放 + 15s keepalive)。
- **结果**:`GET /api/results`、`POST /api/aggregate`(跨节点 email+apiKey 去重)、`POST /api/results/delete|clear`、`GET /download`。
- **按需工具**:`POST /api/accounts/change-pw`(改邮箱密码)、`POST /api/accounts/get-key`(取新 Key)— 都 spawn Python 工具,密码走 **stdin 不进 argv**。
- **资源/台账**:`/api/cards|accounts|proxies|addresses|adspower|captcha|mailbox|billing|errors|policy|runs|recovery|strategies|engine-configs|advanced|selectors` 各自 CRUD。
- **集群**:`POST /api/register`(子机心跳,TTL 90s)、`POST /api/push`(结果上行)。
- **诊断/健康**:`/api/diagnose`、`/api/health|overview|analytics`、`/api/setup/status`。
- **运维**:`POST /api/restart`(一键重启)、`POST /api/system/kill-orphans`(清孤儿 chromedriver)。

## 7. 命令行与编排

**入口**:`run.py`(纯Sel批量)、`hybrid_run.py`(混合)、`hyb_loop.py`(混合自重试到全绑)、`_cardloop.py`(纯Sel加卡循环)。完整清单见 [selenium-e2e/ENTRYPOINTS.md](selenium-e2e/ENTRYPOINTS.md)。

**常用参数**(以 `hybrid_run.py` 为主):

| 参数/开关 | 作用 |
|---|---|
| `--accounts` / `--proxies` / `--op-pw`(统一密码) | 账号文件(`email:邮箱密码`)/ 代理池 / OpenRouter 登录密码 |
| `--concurrency N` | 同跑几个号(窗口自动平铺成网格) |
| `--do-key` / `--do-card` / `--do-purchase` / `--do-changepw` | 各阶段开关 |
| `--real-charge` / `--charge-count N` / `--limit-by-capacity` | 真实充值开关(默认 dry-run)/ 整批真扣帽 / 按卡容量限批 |
| `--max-rotations N` / `--cooldown-hours H` / `--max-reopen N` | 加卡切 IP 次数 / 冷却 / 同环境重开上限 |
| `--no-delete-env` / `--no-resume` / `--proxy-offset K` | 调试留环境 / 不续跑(仍不重建Key不重扣)/ 代理起点偏移 |

**并发护栏**:`--concurrency` 之外,AdsPower 本地 API 全局限频 ~1req/s;`ADS_MAX_LAUNCH`(默认 2)限同时**启动**浏览器数(防 `session deleted`);高级参数 `maxConcurrency` 在 split 模式两引擎各钳一半。

**断点续跑(逐阶段)**:进度落 `state/*progress.json` + `results.jsonl`,每阶段(register/login/key/card/charge/changepw)独立状态机。续跑只补没做完的步:
- 已绑卡 → 跳过加卡;有 Key → 复用绝不重建;上次留了环境 → 重开同环境(免重登/免重绑地址)。

**自动重试失败号**:`AUTO_RETRY_FAILED` 开关,主批后按 `fail_stage` + 恢复策略只重跑可救的失败号(瞬时类),永久失败(NOT_ALLOWED/坏邮箱)不重试,复用 prior 防重取 Key/重扣。

**Node↔Python 契约(改动勿破坏)**:子进程 **stdout 末行才是 JSON 结果,所有日志走 stderr**;`results.jsonl` 用 `os.write(O_APPEND)+fsync` 原子整行追加(多 job 并发不撕裂)。

## 8. 运营工具(`selenium-e2e/tools/`)

都从 `selenium-e2e/` 下 `python tools/<名>.py` 跑。无需启动整条流水线,随时可用。

| 工具 | 用途 |
|---|---|
| `status.py` | **运营仪表盘**:进度/冷却队列倒计时/被拒/卡池余量(够不够+几个BIN)/今日各 BIN 战绩(支持 `--watch`) |
| `cards_watch.py` / `watch_results.py` | 实时盯卡池 / 实时盯结果流 |
| `card_capacity.py` / `卡查询.py` | 看每卡容量 / 查单卡在池中的全字段 + 是否会被选中 |
| `import_cards.py` / `disable_cards.py` / `reactivate_cards.py` / `block_bin.py` | 导卡 / 禁卡 / 救活误禁卡 / 拉黑整个 BIN(都带跨进程文件锁) |
| `cards_failed.py` / `flag_accounts.py` | 列绑卡失败的卡(完整号写 `_failed_cards.log`)/ 标没绑上卡的问题号 |
| `proxy_score.py` | 代理质量评分(0–100,按加卡战绩分档/退役) |
| `cleanup_envs.py` | 环境兜底 GC(删孤儿 `hyb-*`,三护栏防误删续跑环境) |
| `verify_card.py` / `addcard.py` / `purchase.py` | 只读校验已绑卡 / 纯 Sel 端到端加卡 / 纯 Sel 充值($5+,真扣) |
| `fixc_probe.py` / `fingerprint_check.py` | Fix C A/B 探针(原生CDP vs chromedriver)/ 指纹自测(当验收镜子,非生成器) |
| `change_mailbox_pw.py` / `get_new_key.py` | **被 Web 调用**的 stdin-JSON 工具:批量改邮箱密码 / 已注册号登录建新 Key |
| `report_b3d.py` | 批次最终报表(耗时/绑成率/两引擎对比/hcaptcha 实况/坏邮箱) |

> **stdin-JSON 工具铁律**:凭据只走 JSON stdin 绝不进 argv;退出码恒 0,逐行 JSON 的 `ok` 字段定成败;逐行加锁防破行。

## 10. 集群与跨平台部署

- **集群**:中心机在 `config.json` 配 `cluster.hosts`(或子机配 `centralUrl` **自动注册 + 30s 心跳推送**);结果聚合页自动汇总全节点。派发接口按节点数分片 accounts/proxies/cards,**下发时冻卡防双扣**。SSRF 防御:派发/聚合目标必须在集群白名单内。详见 [docs/集群部署手册.md](docs/集群部署手册.md)。
- **Linux 服务器**:推荐 headless;需有头用 Xvfb 虚拟显示 + pm2 常驻。详见 [docs/DEPLOY.md](docs/DEPLOY.md)。
- **macOS**:Apple Silicon 上 byte-patch chromedriver 后会被 SIGKILL → 代码自动 ad-hoc 重签名;多屏需显式 `SCREEN_W/SCREEN_H`。详见 [docs/macOS-跨平台运行.md](docs/macOS-跨平台运行.md)。

## 11. 关键约束与坑(改动前务必知道)

1. **共享卡池字段两端一致**:`status='disabled'` 不是 `'declined'`、`cooldownUntil`/`declineCount` 等语义 Node/Python 必须同口径(见 [billing/README.md](billing/README.md))。
2. **实时与最终成功口径必须相等**:`parseResultLine`(实时计数)与 `isSuccessRow`(最终聚合)都按 steps 逐级 gate,否则"实时 N 个、最终 M 个"数据撕裂。
3. **账单状态单调护栏**:新状态等级 ≥ 旧才覆盖,绝不把已有 card-bound/success 降级 → 防续跑误判未达标重绑/重扣。
4. **充值多层防重**:checkpoint `stages.charge` + `results.jsonl` 的 `charged>0`/`purchase=success` + per-card 原子预留(reserve/commit/release,10min 自动 reap)。
5. **取 Key 失败的号绝不再加卡/充值/改密** —— 避免"无 Key 的废号"白绑卡浪费卡池/BIN 额度。
6. **改密只碰邮箱密码账本,绝不动 account-store 的续跑/恢复字段**(污染会导致登不回去/重复扣款)。
7. **`web/package.json` 绝不加 `type:module`**(`server.js` 是 CJS);改前端必须 `npm run build` 重出 `public/assets` 再重启服务。
8. **反检测边界**:Fix C 的 CDP 注入、Turnstile wrapper、滑块/指纹逻辑都是已验证闭环,**非必要不重写**;碰这些先看 `memory/` 里的 `openrouter-*` 经验记录。
9. **真机才是终审**:离线测试只能验到协议/数据流/类型边界;选择器、Stripe 嵌套、factor-two、滑块标定等必须真站跑过才算端到端确认。
