# 代理预检链边界检查

对应目录：
- `D:\playwright\shared-proxy-precheck`

这个文档只做一件事：
**把代理预检链该做什么、不该做什么、和正式注册链怎么切、和 runner 怎么分层，先钉死。**

---

# 一、代理预检链是什么

代理预检链不是注册主链的一部分业务阶段。
它的职责是：

1. 先检查当前代理是否值得继续使用
2. 先判断代理是否具备打开目标站点并达到首屏可用态的能力
3. 在正式注册开始前，给出结构化 success / failure / unknown 结果

---

# 二、代理预检链负责什么

代理预检链负责：
- 代理连通性检查
- 代理基础网络健康检查
- 站点入口可达性检查
- 站点首屏/业务首屏 ready 检查
- 统一 success / failure / unknown 收口
- 失败分类

---

# 三、代理预检链不负责什么

代理预检链不负责：
- 邮箱密码提交
- 验证码处理
- 生日/资料填写
- 登录后用户态确认
- account delivery
- runner 的代理池管理策略
- 外部写库
- 消息通知

---

# 四、和正式注册链怎么切

## 代理预检链终点
代理预检链成功定义是：
- 已确认当前代理具备进入正式注册链的最低可用条件
- 已确认可以把代理交给正式注册主链继续使用

## 正式注册链起点
正式注册链起点是：
- 当前代理已经通过预检
- 可以开始 entry / credential / verification 等业务阶段

---

# 五、一句话总结

代理预检链只做一件关键事：
**把“这个代理是否值得交给正式注册链使用”这件事确认干净。**
