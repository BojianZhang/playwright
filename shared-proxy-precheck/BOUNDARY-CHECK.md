# 代理预检测速链边界检查

对应目录：
- `D:\playwright\shared-proxy-precheck`

这个文档只做一件事：
**把代理预检测速链该做什么、不该做什么、和正式注册链怎么切、和 runner 怎么分层，先钉死。**

---

# 一、代理预检测速链是什么

代理预检测速链不是注册主链的一部分业务阶段。
它的职责是：

1. 先检查当前代理是否具备基础网络连通性
2. 先检查当前代理的出口 IP 是否可读取
3. 先检查当前代理访问即梦相关目标时是否可达、响应是否合理
4. 在正式注册开始前，给出结构化 success / weak / failure / unknown 结果

---

# 二、代理预检测速链负责什么

代理预检测速链负责：
- 代理基础连通性检查
- 代理出口 IP 检查
- 即梦主目标检查
- 即梦副目标检查
- 统一 success / weak / failure / unknown 收口
- 失败分类与分级

---

# 三、代理预检测速链不负责什么

代理预检测速链不负责：
- Playwright 打开页面
- 首页 UI ready 检查
- 业务首屏判断
- 邮箱密码提交
- 验证码处理
- 生日/资料填写
- 登录后用户态确认
- account delivery
- runner 的代理池管理策略
- 外部写库
- 消息通知

---

# 四、和正式注册链怎么切

## 代理预检测速链终点
代理预检测速链成功定义是：
- 已确认当前代理具备进入正式注册链的最低网络可用条件
- 已确认可以把代理交给正式注册主链继续使用

## 正式注册链起点
正式注册链起点是：
- 当前代理已经通过预检测速
- 可以开始 entry / credential / verification 等业务阶段

---

# 五、一句话总结

代理预检测速链只做一件关键事：
**把“这个代理从网络与即梦目标访问层面看是否值得交给正式注册链使用”这件事确认干净。**
